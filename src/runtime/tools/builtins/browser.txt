Controls a browser or desktop environment to navigate websites, interact with UI elements, extract data, and capture screenshots.

Use this tool when you need to:
- Navigate to websites and interact with web applications
- Fill forms, click buttons, or perform user actions
- Extract structured data from web pages
- Capture screenshots of web pages or desktop
- Execute complex multi-step browser workflows

This tool connects to the Allternit Computer Use service which provides browser automation through Playwright, browser-use (LLM-powered), and Chrome DevTools Protocol adapters.

Available actions:
- execute: Execute a high-level goal using the browser-use adapter (LLM-powered automation)
- goto: Navigate to a specific URL
- click: Click an element by selector or description
- fill: Fill an input field with text
- extract: Extract structured data from the page
- screenshot: Capture a screenshot of the current page
- inspect: Get page structure and available elements

The tool returns a normalized result envelope including:
- status: completed, failed, or cancelled
- summary: Text summary of what was accomplished
- artifacts: Screenshots, downloads, or extracted data
- receipts: Action receipts for audit trail
- session_id: The Computer Use session identifier
- adapter_id: Which adapter was used (playwright, browser-use, cdp)

Important notes:
- This tool requires user permission for each invocation
- Screenshots are returned as file attachments
- Session state persists across calls within the same GIZZI session
- For complex workflows, use the 'execute' action with a clear goal
- For precise control, use individual actions (goto, click, fill)
- Always provide clear, specific selectors or descriptions for elements

Example workflow:
1. Use goto with a URL to navigate to a webpage
2. Use screenshot (optionally with the same URL) to capture what you see
3. The screenshot will be returned as a file attachment you can view
